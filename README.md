# VoIP SIP Analyzer

A simple web application that analyzes VoIP phone call captures (PCAP files) and automatically determines what happened during each call.

Upload a network capture from Wireshark, tcpdump, or another packet capture tool, and the system will identify whether calls were:

* ✅ Answered
* 📞 Missed
* ❌ Rejected
* 🚫 Cancelled
* ⚠️ Failed

Perfect for VoIP engineers, telecom administrators, support teams, and anyone troubleshooting SIP-based phone systems.

---

## What Does It Do?

VoIP SIP Analyzer reads SIP traffic from PCAP or PCAPNG files and converts complex packet data into an easy-to-understand call history.

Instead of manually inspecting hundreds of SIP messages in Wireshark, you can:

* Upload one or many capture files
* View all detected calls
* See call outcomes instantly
* Inspect SIP message flows
* Run validation tests
* Generate call statistics and analytics

---

## Features

### Call Detection & Classification

Automatically detects and classifies calls as:

| Status    | Description                                   |
| --------- | --------------------------------------------- |
| ANSWERED  | The call was successfully connected           |
| MISSED    | The phone rang but nobody answered            |
| REJECTED  | The call was actively declined or busy        |
| CANCELLED | The caller cancelled before ringing completed |
| FAILED    | The call failed because of an error           |

---

### PCAP Analysis

Supports:

* PCAP files

* Single-file uploads
* Bulk uploads (up to 25 files)

Works with captures generated by:

* Wireshark

### Visual Call Inspection

View:

### Analytics Dashboard

Get insights such as:

* Total calls processed

## How It Works

The analyzer examines SIP signaling messages and determines the final outcome of each call.

Examples:

| SIP Flow                      | Result    |
| ----------------------------- | --------- |
| INVITE → 200 OK → BYE         | ANSWERED  |
| INVITE → 180 Ringing → CANCEL | MISSED    |
| INVITE → 486 Busy Here        | REJECTED  |
| INVITE → 603 Decline          | REJECTED  |
| INVITE → CANCEL               | CANCELLED |
| INVITE → 404 or 5xx Error     | FAILED    |

## Configuration

The application works out of the box with default settings.

### Database

Default:

```text
SQLite
```

Data is stored locally and persists across container restarts.

Optional PostgreSQL support:

```env
DATABASE_URL=postgresql+asyncpg://USER:PASSWORD@HOST:5432/DBNAME
```

---

### SIP Parser

Default parser:

```text
Scapy
```

Benefits:

* Pure Python
* No tshark installation required
* Easy deployment
* Reliable SIP parsing

Optional:

```env
SIP_PARSER_BACKEND=pyshark
```

for environments that require tshark-based decoding.

## Requirements

* Docker & Docker Compose

or

## Limitations

* SIP traffic must be unencrypted
* TLS-encrypted SIP is not supported
* SRTP media streams are not analyzed
* Upload limits can be configured through environment variables

---

## API Endpoints

Main API routes:

| Method | Endpoint           | Purpose                  |
| ------ | ------------------ | ------------------------ |
| GET    | /health            | Health check             |
| POST   | /upload-pcap       | Upload a capture         |
| POST   | /upload-pcap/batch | Upload multiple captures |
| GET    | /calls             | List detected calls      |
| GET    | /calls/{id}        | View call details        |
| GET    | /analytics         | Call statistics          |
| POST   | /replay-test       | Validation testing       |

Interactive API documentation:

```text
http://localhost:8000/docs
```
